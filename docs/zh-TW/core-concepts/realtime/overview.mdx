當您的應用程式需要在不重新整理頁面的情況下進行更新時，使用 InsForge Realtime。用戶端訂閱通道，如 `order:123` 或 `chat:room-1`，然後透過 WebSocket 接收資料庫變更、廣播和狀態更新。通道也可以將相同的訊息扇出到 webhook URL，當另一項服務應接收事件時。

## 下一步

- 設定 [CLI](/quickstart) 以連結您的專案。
- 在即時儀表板中建立通道。
- 使用 [TypeScript SDK 參考](/sdks/typescript/realtime) 進行用戶端訂閱。
- 當另一項服務需要相同的事件流時，向通道新增 webhook URL。
