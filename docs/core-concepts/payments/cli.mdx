The Payments CLI is the admin and agent control surface for [Stripe](https://stripe.com) setup: keys, sync, catalog, projections. Runtime checkout belongs in app code with the [TypeScript Payments SDK](/sdks/typescript/payments).

## Concepts

Examples use `npx @insforge/cli` (don't install globally). Pass `--json` for scripts and `--yes` for non-interactive destructive commands. Refunds and disputes belong in the Stripe Dashboard.

## Usage

Run `status` to see key, account, sync, and webhook state. Setting a key validates with Stripe, records account identity, configures the webhook, and runs sync.

```bash
npx @insforge/cli payments status
npx @insforge/cli payments config set test sk_test_xxx
npx @insforge/cli payments config set live sk_live_xxx
```

Sync pulls products, prices, customers, and subscriptions. Stripe is the source of truth; sync overwrites local drift.

```bash
npx @insforge/cli payments sync
npx @insforge/cli payments catalog --environment test
```

## Specific usage cases

Create a product and recurring price. Use idempotency keys so retries don't duplicate.

```bash
npx @insforge/cli payments products create \
  --environment test --name "Pro Plan" \
  --idempotency-key "product:pro"

npx @insforge/cli payments prices create \
  --environment test --product prod_123 \
  --currency usd --unit-amount 1900 --interval month \
  --idempotency-key "price:pro:monthly"
```

Stripe prices are immutable for amount, currency, and cadence. Archive and recreate to change those. Products with prices cannot be deleted; use `products update --active false`.

```bash
npx @insforge/cli payments prices archive price_123 --environment test
```

Retry webhook setup after deploying to a public URL:

```bash
npx @insforge/cli payments webhooks configure test
```

Read-only views for debugging. Don't expose to end users.

```bash
npx @insforge/cli payments subscriptions --environment test
npx @insforge/cli payments history --environment test --subject-type team --subject-id team_123
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Payments are not available on this backend` | Upgrade the self-hosted backend or ask the project admin to enable payments |
| No catalog after configuring keys | Run `payments sync`; confirm the right environment key |
| Webhook setup fails on localhost | Deploy to a public URL or use the [Stripe CLI](https://docs.stripe.com/stripe-cli) |
| Portal returns no customer mapping | Have the customer complete Checkout first, then retry |
