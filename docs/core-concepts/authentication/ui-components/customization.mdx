The Insforge UI packages provide a powerful **hierarchical appearance system** that allows you to customize every aspect of authentication componentsâ€”from top-level containers to nested elements like form fields and buttons.

<Note>
**Framework Support**: This customization system works identically across all Insforge UI packages:
- `@insforge/nextjs` (Next.js)
- `@insforge/react` (Vite + React Router)
</Note>

## Hierarchical Appearance System

The hierarchical appearance system lets you style nested components through a JSON-like structure, giving you complete control over every element.

**Example:**
```tsx
<SignIn
  appearance={{
    card: "bg-white shadow-2xl",
    header: {
      title: "text-3xl font-bold",
      subtitle: "text-gray-600"
    },
    form: {
      emailField: {
        input: "border-blue-500"
      }
    },
    button: "bg-blue-600"
  }}
/>
```

## Complete Appearance Structure

### SignIn / SignUp Components

Full appearance hierarchy for authentication forms:

```tsx
import { SignIn } from '@insforge/nextjs'; // or '@insforge/react'

<SignIn
  appearance={{
    container?: string;              // Outermost wrapper
    card?: string;                   // Inner card box
    
    header?: {
      container?: string;            // Header wrapper
      title?: string;                // Title text
      subtitle?: string;             // Subtitle text
    };
    
    errorBanner?: string;            // Error message banner
    
    form?: {
      container?: string;            // Form wrapper
      
      emailField?: {
        container?: string;          // Email field wrapper
        label?: string;              // Email label
        input?: string;              // Email input
      };
      
      passwordField?: {
        container?: string;          // Password field wrapper
        label?: string;              // Password label
        input?: string;              // Password input
        forgotPasswordLink?: string; // Forgot password link (SignIn only)
        
        strengthIndicator?: {        // Password strength (SignUp only)
          container?: string;
          requirement?: string;
        };
      };
    };
    
    button?: string;                 // Submit button
    
    link?: {
      container?: string;            // Link section wrapper
      text?: string;                 // Link description text
      link?: string;                 // Actual link element
    };
    
    divider?: string;                // "or" divider
    
    oauth?: {
      container?: string;            // OAuth buttons wrapper
      button?: string;               // Individual OAuth button
    };
  }}
/>
```

### UserButton Component

```tsx
<UserButton
  appearance={{
    button?: string;        // Main button
    name?: string;          // User name text
    email?: string;         // User email text
    dropdown?: string;      // Dropdown menu
  }}
/>
```

## Text Customization

All components support complete text customization:

### SignIn Component

```tsx
<SignIn
  title="Welcome Back!"
  subtitle="We're happy to see you again"
  emailLabel="Your Email Address"
  emailPlaceholder="you@company.com"
  passwordLabel="Your Password"
  passwordPlaceholder="Enter your password"
  submitButtonText="Login Now"
  loadingButtonText="Signing you in..."
  forgotPasswordText="Forgot password?"
  signUpText="New to our platform?"
  signUpLinkText="Create an account"
  dividerText="or continue with"
/>
```

### SignUp Component

```tsx
<SignUp
  title="Create Account"
  subtitle="Get started with your free account"
  emailLabel="Email Address"
  passwordLabel="Choose Password"
  submitButtonText="Create Account"
  loadingButtonText="Creating your account..."
  signInText="Already have an account?"
  signInLinkText="Sign in"
  dividerText="or sign up with"
/>
```

### UserButton Component

```tsx
<UserButton
  mode="detailed"  // or "simple"
  afterSignOutUrl="/"
  appearance={{
    button: "hover:bg-gray-100 rounded-full p-2",
    name: "text-gray-900 font-semibold",
    email: "text-gray-500 text-sm",
    dropdown: "shadow-xl border border-gray-200"
  }}
/>
```

**Display Modes:**
- `detailed` (default): Shows avatar + name + email
- `simple`: Shows avatar only

## Styling Atomic Components

When building custom forms with atomic components, each supports its own appearance structure:

### AuthContainer

```tsx
<AuthContainer
  appearance={{
    container: "max-w-md mx-auto",
    card: "bg-white shadow-2xl rounded-2xl p-8"
  }}
>
  {/* Your form content */}
</AuthContainer>
```

### AuthHeader

```tsx
<AuthHeader
  title="Welcome Back"
  subtitle="Sign in to continue"
  appearance={{
    container: "mb-8 text-center",
    title: "text-3xl font-bold text-gray-900",
    subtitle: "text-gray-600 mt-2"
  }}
/>
```

### AuthFormField

```tsx
<AuthFormField
  id="email"
  type="email"
  label="Email Address"
  placeholder="you@example.com"
  appearance={{
    container: "mb-4",
    label: "text-gray-700 font-semibold mb-2",
    input: "border-2 border-gray-300 focus:border-blue-500 rounded-lg px-4 py-3"
  }}
/>
```

### AuthPasswordField

```tsx
<AuthPasswordField
  id="password"
  label="Password"
  emailAuthConfig={config}
  showStrengthIndicator
  forgotPasswordLink={{
    href: "/forgot-password",
    text: "Forgot password?"
  }}
  appearance={{
    container: "mb-4",
    label: "text-gray-700 font-semibold mb-2",
    input: "border-2 border-gray-300 focus:border-blue-500 rounded-lg px-4 py-3"
  }}
/>
```

### AuthSubmitButton

```tsx
<AuthSubmitButton
  isLoading={loading}
  confirmed={success}
  className="w-full bg-blue-600 hover:bg-blue-700 text-white font-bold py-3 rounded-lg"
>
  Sign In
</AuthSubmitButton>
```

### AuthOAuthProviders

```tsx
<AuthOAuthProviders
  providers={['google', 'github', 'discord']}
  onClick={handleOAuth}
  loading={oauthLoading}
  appearance={{
    container: "space-y-3",
    button: "border-2 border-gray-200 hover:bg-gray-50 rounded-lg"
  }}
/>
```

**Adaptive Layout:**
- 1 provider: Full width with full text ("Sign in with Google")
- 2 providers: Two columns with short text ("Google", "GitHub")
- 3+ providers: Three columns with icons only

### AuthDivider

```tsx
<AuthDivider 
  text="or continue with"
  className="my-6 text-gray-500"
/>
```

### AuthLink

```tsx
<AuthLink
  text="Don't have an account?"
  linkText="Sign up"
  href="/sign-up"
  appearance={{
    container: "mt-6 text-center",
    text: "text-gray-600",
    link: "text-blue-600 hover:underline font-semibold ml-1"
  }}
/>
```

## Framework-Specific Notes

### Next.js

```tsx
// app/sign-in/page.tsx
'use client';

import { SignIn } from '@insforge/nextjs';

export default function SignInPage() {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      <SignIn
        appearance={{
          // Your customization here
        }}
      />
    </div>
  );
}
```

### React (Vite, React Router)

```tsx
// src/pages/SignInPage.tsx
import { SignIn } from '@insforge/react';

export default function SignInPage() {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      <SignIn
        appearance={{
          // Your customization here
        }}
      />
    </div>
  );
}
```

<Note>
The appearance API is identical across all frameworks. Styles defined in one framework will work in another without modification.
</Note>

## TypeScript Support

Full TypeScript support with exported appearance types:

```tsx
import type {
  SignInAppearance,
  SignUpAppearance,
  ForgotPasswordAppearance,
  ResetPasswordAppearance,
  UserButtonAppearance,
} from '@insforge/nextjs'; // or '@insforge/react'

const myCustomAppearance: SignInAppearance = {
  card: "bg-white shadow-xl",
  header: {
    title: "text-3xl font-bold",
    subtitle: "text-gray-600"
  },
  // TypeScript ensures all keys are valid
};

<SignIn appearance={myCustomAppearance} />
```

## Utility Functions

### cn() - Class Merging Utility

Merge Tailwind classes intelligently (combines `clsx` and `tailwind-merge`):

```tsx
import { cn } from '@insforge/nextjs'; // or '@insforge/react'

const className = cn(
  'px-4 py-2',
  'bg-blue-500',
  isActive && 'bg-blue-700',
  'hover:bg-blue-600'
);
// Resolves conflicting classes intelligently
```
