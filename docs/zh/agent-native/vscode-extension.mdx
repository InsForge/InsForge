## 概览

InsForge VS Code 扩展：

- 让您登入 InsForge（OAuth + PKCE）
- 让您选择组织/项目
- 为您的 AI 客户端安装 InsForge MCP

它通过您的项目的 API 密钥和 API 基础 URL 启动 MCP 安装程序（`npx @insforge/install`）。

## 链接

- [VS Code 市场](https://marketplace.visualstudio.com/items?itemName=InsForge.insforge)
- [开放 VSX](https://open-vsx.org/extension/InsForge/insforge)

## 安装

1. 打开 VS Code
2. 去**扩展**
3. 搜索**InsForge**
4. 安装由**insforge**发布的扩展

## 使用扩展

<Steps>
  <Step title="打开 InsForge 面板">
    点击活动栏（左侧边栏）中的**InsForge**图标。
  </Step>

  <Step title="登入">
    点击**用 InsForge 登入**并在您的浏览器中完成登入流程。

    <Note>回调是 <code>http://127.0.0.1:54321/callback</code>。</Note>
  </Step>

  <Step title="选择项目">
    从 InsForge 树视图中选择组织和项目，或从命令面板运行 <code>InsForge: Select Project</code>。
  </Step>

  <Step title="安装 MCP">
    运行 <code>InsForge: Install MCP</code>（或右键点击项目并选择**安装 MCP**），然后选择您要配置的 AI 客户端。

    安装程序在 VS Code 终端中运行。
  </Step>
</Steps>

## 支持的 AI 客户端

扩展支持以下 MCP"客户端"（传递给 `@insforge/install` 的值）：

| 客户端 | 典型配置位置 |
|---|---|
| `cursor` | `~/.cursor/mcp.json` |
| `claude-code` | 工作区中的 `.mcp.json`（项目本地） |
| `copilot` | 工作区中的 `.vscode/mcp.json`（项目本地） |
| `windsurf` | `~/.codeium/windsurf/` |
| `cline` | Cline VS Code 扩展配置 |
| `roocode` | Roo Code VS Code 扩展配置 |
| `codex` | OpenAI Codex CLI 配置 |
| `trae` | Trae IDE 配置 |
| `qoder` | Qoder IDE 配置 |

如果您的客户端未列出，请手动运行安装程序（请参阅<a href="/quickstart">快速入门</a>）。

## 注意

- **项目本地安装**：`claude-code` 和 `copilot` 需要打开的工作区文件夹。
- **端口在使用中**：释放 `54321` 并重试登入。
