## Descripción general

El servidor InsForge MCP proporciona a su asistente de codificación IA acceso directo a su backend: consultas de base de datos, gestión de esquemas, almacenamiento y más.

Siga las instrucciones a continuación para su cliente de IA. ¿No ve su herramienta? Explore el [directorio de agentes](https://insforge.dev/agents) para obtener la lista completa de integraciones probadas.

## Requisitos previos

- Un **asistente de codificación de IA** (Cursor, Claude Code, GitHub Copilot, etc.)
- Un **proyecto InsForge** - [cree uno aquí](https://insforge.dev) si aún no lo ha hecho

## MCP remoto

La forma más rápida de comenzar es con MCP remoto. Un único comando intenta instalar y manejará la autenticación.

Ejecute el siguiente comando en su terminal:

```bash
npx add-mcp https://mcp.insforge.dev/mcp
```

<Note>
  El comando `add-mcp` funciona para la mayoría de los agentes de codificación, pero no todos. Actualmente admite:
  Claude Code, Claude Desktop, Codex, Cursor, Gemini CLI, Goose, GitHub Copilot, OpenCode, VS Code, Zed.
</Note>

Después de ejecutar el comando, el sistema generará automáticamente un archivo de configuración para su agente de codificación.

### Conectar y vincular proyecto

Algunos clientes MCP le pedirán automáticamente que inicie sesión durante la configuración, mientras que otros pueden requerir autenticación manual o pasos de vinculación de proyecto.

### Verificar instalación

Para verificar la conexión, inicie una nueva sesión de chat en su asistente de codificación de IA y envíe este aviso:

```
I'm using InsForge as my backend platform, call InsForge MCP's fetch-docs tool to
learn about InsForge instructions.
```

Si la conexión es exitosa, verá el indicador de estado **MCP conectado** en su panel de InsForge (esquina superior derecha).

![MCP conectado](/images/mcp-setup/mcp-connected.png)

## Solución de problemas

<Accordion title="Los cambios no tienen efecto">
  La mayoría de los clientes de IA requieren un reinicio completo después de cambios en la configuración de MCP. Cierre y vuelva a abrir la aplicación.
</Accordion>
