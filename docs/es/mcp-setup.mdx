## Resumen

El servidor MCP de InsForge le da a tu asistente de codificación con IA acceso directo a tu backend: consultas de base de datos, gestión de esquemas, almacenamiento y más.

Sigue las instrucciones a continuación para tu cliente de IA. ¿No ves tu herramienta? Consulta el [directorio de agentes](https://insforge.dev/agents) para ver la lista completa de integraciones probadas.

## Requisitos previos

- Un **asistente de codificación con IA** (Cursor, Claude Code, GitHub Copilot, etc.)
- Un **proyecto de InsForge** - [crea uno aquí](https://insforge.dev) si aún no tienes uno

## MCP remoto

La forma más rápida de empezar es con MCP remoto. Un solo comando intenta la instalación e intentará gestionar la autenticación y la vinculación del proyecto, pero puede que aun así te pida pasos manuales de autenticación o vinculación de proyecto si el flujo automático no puede completarse.

Ejecuta el siguiente comando en tu terminal:

```bash
npx add-mcp https://mcp.insforge.dev/mcp
```

<Note>
  El comando `add-mcp` funciona para la mayoría de los agentes de codificación, pero no para todos. Actualmente admite:
  Claude Code, Claude Desktop, Codex, Cursor, Gemini CLI, Goose, GitHub Copilot, OpenCode, VS Code, Zed.
</Note>

Después de ejecutar el comando, el sistema generará automáticamente un archivo de configuración para tu agente de codificación.

### Conectar y vincular el proyecto

Algunos clientes MCP te pedirán iniciar sesión automáticamente durante la configuración, mientras que otros pueden requerir pasos manuales de autenticación o vinculación de proyecto si el flujo automático no puede completarse. De cualquier forma, la autenticación abrirá una ventana del navegador donde podrás iniciar sesión en tu cuenta de InsForge y conceder acceso de organización/proyecto al cliente MCP.

<AccordionGroup>
  <Accordion title="Cursor">
    Encontrarás el archivo de configuración en `.cursor/mcp.json` como se muestra a continuación:

    ```json
    {
      "mcpServers": {
        "insforge": {
          "url": "https://mcp.insforge.dev/mcp"
        }
      }
    }
    ```

    En el menú **Preferences → Cursor Settings → Tools & MCP**, deberías ver que el servidor MCP de InsForge se agregó correctamente.
    El servidor MCP de InsForge está en el estado "Needs authentication"; puedes hacer clic en el botón `connect` para autenticarte.
  </Accordion>

  <Accordion title="Claude Code">
    Después de la instalación, encontrarás el archivo de configuración en `.mcp.json` como se muestra a continuación:
    ```json
    {
      "mcpServers": {
        "insforge": {
          "type": "http",
          "url": "https://mcp.insforge.dev/mcp"
        }
      }
    }
    ```
    Luego, en una terminal normal (no en la extensión del IDE), ejecuta:
    ```sh
    claude /mcp
    ```
    Selecciona el servidor "insforge" y luego "Authenticate" para iniciar el flujo de autenticación.

  </Accordion>

  <Accordion title="GitHub Copilot">

  <Accordion title="Codex">
    <Note>
      Si es la primera vez que usas Codex, es posible que debas habilitar la función rmcp. Para ello, agrega lo siguiente a tu archivo `~/.codex/config.toml`:
      ```toml
      [beta]
      ```

    Después de la instalación, encontrarás el archivo de configuración en `.codex/config.toml` como se muestra a continuación:
    ```toml

    url = "https://mcp.insforge.dev/mcp"
    ```
    Luego autentícate:
    ```sh
    codex mcp login insforge
    ```
    Finalmente, ejecuta /mcp dentro de Codex para verificar la autenticación.

    Alternativamente, puedes ejecutar el siguiente comando para instalar el servidor MCP:
    ```sh
    codex mcp add insforge --url https://mcp.insforge.dev/mcp
    ```
  </Accordion>

  <Accordion title="Cline">
    Puedes instalar el servidor MCP de InsForge en tu **Cline** ejecutando el siguiente comando en tu terminal:
    ```bash
    npx -y @smithery/cli mcp add devel/insforge --client cline
    ```
    Alternativamente, puedes seguir la [documentación oficial de Cline](https://docs.cline.bot/mcp/connecting-to-a-remote-server) para agregar el servidor MCP de InsForge con los siguientes parámetros:
    - Server Name: `insforge`
    - Server URL: `https://mcp.insforge.dev/mcp`
    - Transport Type: `Streamable HTTP`

    Después de agregar el servidor, verás un mensaje de error que solicita autenticarte. Haz clic en el botón "Authenticate" que aparece para autenticarte.

  </Accordion>

  <Accordion title="Windsurf">
    Agrega esta configuración a `~/.codeium/windsurf/mcp_config.json`:
    ```json
    {
      "mcpServers": {
        "insforge": {
          "command": "npx",
          "args": [
            "-y",
            "mcp-remote",
            "https://mcp.insforge.dev/mcp"
          ]
        }
      }
    }
    ```
    Windsurf actualmente no admite servidores MCP remotos mediante transporte HTTP. Debes usar el paquete mcp-remote como proxy.
  </Accordion>

</Accordion>
