InsForge proporciona diez productos, cada uno expuesto a través de la misma superficie REST, la misma autenticación JWT, el mismo servidor MCP y los mismos SDK.

<CardGroup cols={2}>
  <Card title="Base de datos" icon="database" href="/core-concepts/database/overview">

    Cada tabla se convierte en una superficie REST y SDK escrita.
  </Card>

  <Card title="Autenticación" icon="shield" href="/core-concepts/authentication/overview">
    Correo electrónico y contraseña, sin contraseña, OAuth e inicios de sesión móviles a través de una API. Sesiones JWT, seguridad a nivel de fila, modo servidor OAuth.
  </Card>

  <Card title="Almacenamiento" icon="folder" href="/core-concepts/storage/overview">

  </Card>

  <Card title="Tiempo real" icon="tower-broadcast" href="/core-concepts/realtime/overview">

## Cómo encajan

Cada producto comparte la misma identidad de autenticación, las mismas políticas de seguridad a nivel de fila y la misma superficie MCP para agentes de IA. Un usuario que inició sesión a través de **Autenticación** obtiene un JWT que las capas de **Base de datos**, **Almacenamiento**, **Tiempo real** y **Funciones de borde** utilizan para aplicar las mismas reglas de acceso.
