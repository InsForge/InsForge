Utilice InsForge Realtime cuando su aplicación necesite actualizarse sin una actualización de página. Los clientes se suscriben a canales como `order:123` o `chat:room-1`, luego reciben cambios de base de datos, transmisiones y actualizaciones de presencia a través de WebSockets. Los canales también pueden distribuir los mismos mensajes a URL de webhook cuando otro servicio debe recibir el evento.

## Próximos pasos

- Configure el [CLI](/quickstart) para vincular su proyecto.

- Agregue URL de webhook a un canal cuando otro servicio necesite la misma secuencia de eventos.
