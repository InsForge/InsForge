InsForge es la **plataforma de infraestructura en la nube nativa de agentes**. Un agente de codificación proporciona a cualquier aplicación una base de datos Postgres, autenticación, almacenamiento, funciones perimetrales, computación, alojamiento y una puerta de enlace de modelo de IA, impulsada de extremo a extremo a través de un CLI. Es de código abierto (Apache-2.0).

Su servidor MCP y CLI exponen esa infraestructura como contexto estructurado y legible por máquinas, para que los agentes de codificación de IA puedan planificar y ejecutar operaciones de forma autónoma dentro de permisos limitados. El cambio en el rol del humano: con plataformas tradicionales **implementa y ejecuta**; con InsForge **define intención y revisa**.

## InsForge vs. Supabase

Supabase es un backend de código abierto basado en Postgres construido para desarrolladores humanos. InsForge comparte la base de Postgres pero está construido para agentes de codificación de IA como operadores de primera clase.

| Dimensión | Supabase | InsForge |
|---|---|---|
| Operador principal | Desarrolladores humanos | Agentes de codificación de IA |
| Modelo de configuración | UI explícita, SQL, CLI | Operaciones ejecutadas por agentes |
| Diseño de permisos | Acceso gestionado por desarrollador | Autonomía de agente limitada |
| Contexto del sistema | Implícito, mantenido por desarrollador | Estructurado, legible por máquinas (MCP) |
| Pagos | A través del código de la aplicación | Integración Stripe integrada |
| Acceso al modelo de IA | Conexión de modelo externo | Puerta de enlace de modelo integrada |
| Seguridad | Implementación por integración | Valores predeterminados listos para producción |
| Preparación del agente | Requiere conexión manual | Operable de extremo a extremo |

## InsForge vs. Firebase

Firebase se enfoca en prototipos rápidos en NoSQL orientado a documentos con operadores humanos. InsForge se enfoca en codificación agentica en Postgres relacional.

| Dimensión | Firebase | InsForge |
|---|---|---|
| Base de datos | NoSQL orientada a documentos | Postgres relacional |
| Modelo de datos | JSON sin esquema | Tablas estructuradas con relaciones |
| Combinaciones | No soportado de forma nativa | Combinaciones y consultas nativas |
| Transacciones | Compatible con limitaciones | Transacciones ACID completas |
| Auto-alojamiento | No soportado | Docker / auto-hospedable |
| Operador principal | Desarrolladores humanos | Agentes de codificación de IA |
| Contexto del sistema | Implícito, fragmentado | Estructurado, legible por máquinas (MCP) |
| Integración de IA | Configuración externa | Puerta de enlace de modelo integrada |
| Pagos | Integración manual | Primitivo Stripe integrado |
