title: "Telemetría anónima auto-alojada"
description: "Aprenda qué datos de uso anónimos envían los despliegues auto-alojados de InsForge, qué nunca se recopila y cómo optar por no participar usando INSFORGE_TELEMETRY_DISABLED."
---

InsForge recopila telemetría anónima de despliegues auto-alojados para comprender instalaciones activas, adopción de versiones, métodos de implementación y qué servicios opcionales están configurados. Esto ayuda a los mantenedores a priorizar correcciones, avisos de seguridad y trabajo de implementación para el proyecto de código abierto.

La telemetría es opcional. Puede deshabilitarla en cualquier momento.
InsForge no envía esta telemetría desde InsForge Cloud.
Los eventos auto-alojados incluyen una etiqueta de tiempo de ejecución aproximada, como producción, desarrollo, prueba, ci o desconocido, para que los mantenedores puedan separar las ejecuciones locales y automatizadas de los despliegues de producción.

## Lo que envía InsForge

InsForge envía un evento `oss_instance_started` cuando se inicia el backend y un evento `oss_heartbeat` aproximadamente una vez cada 24 horas mientras se ejecuta el backend.
Los eventos se envían a través del proxy PostHog de InsForge en `https://b.insforge.dev/capture/` utilizando el formato de API de captura de PostHog.

Cada evento contiene:

- Nombre del evento PostHog: `oss_instance_started` u `oss_heartbeat`
- ID de instalación anónima almacenado localmente en `LOGS_DIR/.insforge-installation-id`, usado como `distinct_id` de PostHog
- Versión de InsForge
- Marca de tiempo del evento
- Modo de alojamiento, como `self-hosted` o `cloud`
- Método de implementación aproximado, como Docker Compose, Railway, Zeabur, Sealos, Dokploy, Kubernetes o desconocido
- Sistema operativo, arquitectura de CPU, versión de Node.js, entorno de ejecución y si el proceso parece estar ejecutándose en CI
- Categoría de backend de almacenamiento: sistema de archivos local, S3 o compatible con S3
- Banderas booleanas para si las características opcionales están configuradas: despliegues de sitios, funciones, computación y OpenRouter

## Lo que InsForge nunca envía

La telemetría no incluye:

- Valores de variables de entorno
- Claves API, secretos JWT, contraseñas, secretos OAuth o secretos de pago
- Contenidos de base de datos, esquemas, nombres de tablas o recuentos de filas
- Registros, seguimientos de pilas de errores, contenidos de archivos o rutas de archivos
- Nombres de proyectos, dominios, nombres de cubetas, direcciones de correo electrónico o datos de usuario

## Deshabilitar telemetría

Establezca `INSFORGE_TELEMETRY_DISABLED=1` en su entorno y reinicie el backend.
Para despliegues de Docker Compose, agregue la misma línea a su archivo `.env`:

```bash
INSFORGE_TELEMETRY_DISABLED=1
```

Los fallos de red se registran e ignoran para que la telemetría nunca bloquee el inicio o las solicitudes normales.
